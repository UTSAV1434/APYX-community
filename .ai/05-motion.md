# APYX Motion System

## Purpose

Motion is not decoration.

Motion should improve usability, guide attention, communicate hierarchy, and create memorable interactions.

Every animation must have a purpose.

Never animate just because it looks cool.

---

# Motion Principles

Motion should feel:

- Smooth
- Intentional
- Premium
- Fast
- Responsive
- Elegant

Avoid:

- Bouncy animations
- Excessive rotations
- Flashy effects
- Long delays
- Random motion

---

# Animation Curves

Use smooth easing.

Prefer ease-out for entrances.

Ease-in for exits.

Avoid linear animations.

---

# Scroll Behaviour

The website should use smooth scrolling.

Scrolling should feel fluid.

No abrupt jumps.

Large sections should transition naturally.

---

# Scroll Storytelling

Marketing pages should tell a story while scrolling.

Sections should reveal progressively.

Each section should feel connected.

Content should never suddenly appear.

# Card Animations

Cards should respond to hover.

Hover should include

- Lift
- Shadow
- Border highlight
- Soft glow (where appropriate)

Cards should never rotate dramatically.

# Cursor Behaviour

---

# Accessibility

Respect prefers-reduced-motion.

Disable large animations when users request reduced motion.

Maintain usability without motion.

---

# Performance

Animations should never block rendering.

Prefer GPU accelerated transforms.

Avoid layout shifts.

Maintain 60fps whenever possible.

---

# Animation Libraries

Preferred

- Framer Motion
- GSAP (only when necessary)

Avoid multiple animation systems for the same interaction.

Maintain consistency across the project.

---

# Design Philosophy

Motion should create delight without distraction.

Every animation must improve the user experience.

Users should remember the experience, not the animation itself.

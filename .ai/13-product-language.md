# APYX Product Language

Version: 1.0

---

# Purpose

This document defines the writing style, communication principles, and personality of APYX.

Every piece of text shown to users must follow these guidelines.

The goal is to ensure every page feels like it was written by the same team.

---

# Brand Personality

APYX should feel:

• Innovative

• Friendly

• Professional

• Motivating

• Curious

• Confident

• Inclusive

Never sound arrogant.

Never sound overly corporate.

Never sound childish.

---

# Communication Style

Write like experienced builders talking to future builders.

Be simple.

Be direct.

Be inspiring.

Avoid unnecessary buzzwords.

Avoid long paragraphs.

Every sentence should provide value.

---

# Tone

Primary Tone

Confident

Helpful

Encouraging

Clear

Professional

---

Secondary Tone

Community-driven

Optimistic

Modern

Friendly

---

# Voice

APYX speaks as one organization.

Never use different personalities on different pages.

Always sound consistent.

---

# Writing Rules

Use active voice.

Keep sentences short.

Avoid unnecessary technical jargon.

Explain concepts simply.

Write for students first.

Never sound robotic.

---

# Headlines

Headlines should be:

Short

Memorable

Action-oriented

Strong

Examples

Build.

Learn.

Innovate.

Create.

Ship.

Grow.

Join the Community.

Discover Opportunities.

---

# Subheadings

Explain the value.

Never repeat the heading.

Keep under two lines.

---

# Call To Action Rules

Every CTA should encourage action.

Preferred Examples

Join APYX

Explore Events

Register Now

View Gallery

Meet the Team

Become a Partner

Apply Today

Download Resources

Start Building

Stay Updated

---

Avoid

Click Here

Submit

Learn More (unless appropriate)

Press Here

Continue

---

# Event Titles

Event names should be memorable.

Examples

AI Build Sprint

CodeCrafters

HackAPYX

Cloud Launch

Future Builders Summit

---

# Event Descriptions

Every event description should answer:

What is it?

Why should someone attend?

Who is it for?

What will they gain?

---

# Announcement Style

Announcements should be:

Clear

Exciting

Timely

Actionable

Always include:

Context

Important information

Next steps

---

# Empty States

Never leave users with blank screens.

Provide:

Friendly message

Helpful explanation

Primary action

Example

"No upcoming events yet.

Stay tuned—we're building something exciting."

---

# Error Messages

Good

"We couldn't load the events.

Please try again."

Bad

"Error 500"

---

# Success Messages

Examples

Registration successful.

Welcome to APYX!

Changes saved successfully.

You're all set.

See you at the event!

---

# Loading Messages

Examples

Loading Events...

Preparing Your Dashboard...

Fetching Resources...

Loading Community Updates...

---

# Forms

Labels should be clear.

Use helper text where necessary.

Validation should explain the problem.

Bad

Invalid Input

Good

Please enter a valid email address.

---

# Dashboard Language

Professional.

Minimal.

Action-focused.

Examples

Create Event

Publish Announcement

Manage Team

View Analytics

Upload Gallery

---

# Community Language

Always emphasize:

Learning

Building

Sharing

Helping

Collaboration

Innovation

---

# Social Media Tone

Energetic

Positive

Professional

Community-focused

Celebrate achievements.

Recognize contributors.

Encourage participation.

---

# Notification Style

Keep notifications short.

Examples

Event published.

Gallery updated.

Profile saved.

Registration confirmed.

Announcement posted.

---

# Accessibility

Avoid complex vocabulary.

Keep reading level simple.

Use descriptive links.

Avoid ambiguous wording.

---

# AI Content Rules

When AI generates content it must:

Maintain APYX personality.

Avoid generic marketing language.

Never exaggerate achievements.

Never use clickbait.

Write naturally.

Be concise.

Keep information accurate.

---

# Final Principle

Every sentence should answer one question:

"Would this make a student feel excited to be part of APYX?"

If not, rewrite it.
